## 1. 기능 개요

### 1.1 목적

화주의 견적 요청에 대해 포워더들이 경쟁 입찰하고, 화주가 최적의 제안을 선택하는 역경매 시스템입니다.

### 1.2 주요 흐름

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           비딩 시스템 흐름                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   [화주]                        [시스템]                    [포워더]      │
│     │                             │                           │          │
│     │  견적 요청 제출              │                           │          │
│     │ ─────────────────────────▶ │                           │          │
│     │                             │  Bidding 생성 (open)      │          │
│     │                             │ ─────────────────────────▶│          │
│     │                             │                           │ 입찰 제출 │
│     │                             │ ◀─────────────────────────│          │
│     │                             │                           │          │
│     │  입찰 목록 확인              │                           │          │
│     │ ◀───────────────────────── │                           │          │
│     │                             │                           │          │
│     │  낙찰자 선정                 │                           │          │
│     │ ─────────────────────────▶ │                           │          │
│     │                             │  Bidding 상태: awarded    │          │
│     │                             │ ─────────────────────────▶│          │
│     │                             │                           │ 낙찰 알림 │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.3 비딩 번호 체계

비딩 번호는 견적 요청 번호와 동일:

| 구성 | 예시 | 설명 |
|------|------|------|
| **EXSEA00001** | 수출+해상+순번 | Export Ocean |
| **IMAIR00015** | 수입+항공+순번 | Import Air |
| **EXALL00003** | 수출+내륙+순번 | Export Truck |

## 3. 비딩 상태 관리

### 3.1 상태 정의

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          비딩 상태 흐름도                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│                        ┌───────────┐                                     │
│                        │   open    │ ← 생성 시 초기 상태                  │
│                        │  (진행중)  │                                     │
│                        └─────┬─────┘                                     │
│                              │                                           │
│         ┌────────────────────┼────────────────────┐                     │
│         │                    │                    │                     │
│         ▼                    ▼                    ▼                     │
│   ┌───────────┐        ┌───────────┐        ┌───────────┐              │
│   │  awarded  │        │  closed   │        │  expired  │              │
│   │ (낙찰완료) │        │  (마감)   │        │  (자동만료)│              │
│   └───────────┘        └───────────┘        └───────────┘              │
│         │                                                               │
│         ▼                                                               │
│   ┌───────────┐                                                         │
│   │ cancelled │ ← 화주가 취소                                            │
│   │  (취소됨)  │                                                         │
│   └───────────┘                                                         │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 상태 설명

| 상태 | 코드 | 설명 | 전환 조건 |
|------|------|------|-----------|
| **진행중** | `open` | 입찰 접수 중 | - |
| **낙찰완료** | `awarded` | 낙찰자 선정됨 | 화주가 낙찰 처리 |
| **마감** | `closed` | 수동 마감 | 화주가 마감 처리 |
| **만료** | `expired` | 자동 만료 | deadline 경과 |
| **취소** | `cancelled` | 취소됨 | 화주가 취소 |

### 3.3 마감일 (Deadline) 규칙

```
Deadline = ETD - 3 영업일

예시:
- ETD: 2026-01-25 (토)
- Deadline: 2026-01-21 (화) 23:59:59
```

**자동 만료 처리**:
- Deadline 경과 시 자동으로 `expired` 상태 전환
- 스케줄러가 매시간 체크

**상태 아이콘**:
| 상태 | 아이콘 | 색상 |
|------|--------|------|

| awarded | 🏆 | 금색 |

| expired | 🔴 | 빨간색 |

**D-day 뱃지**:
| 조건 | 표시 | 색상 |

```

│   └─────────────────────────────────────────────────────────────────┘   │

├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │

│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 7.2 낙찰 처리 결과

낙찰 확정 시:
1. **Bidding 상태** → `awarded`
2. **낙찰 Bid** → `awarded`
3. **탈락 Bid** → `rejected`
4. **Contract 생성** (준비중)
5. **알림 발송**
   - 낙찰자: "축하합니다! EXSEA00025 건에 낙찰되셨습니다."
   - 탈락자: "EXSEA00025 건의 낙찰자가 선정되었습니다."
